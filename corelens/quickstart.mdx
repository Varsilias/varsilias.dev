This quickstart creates a Corelens instance, writes a structured log, records a metric, creates a span, and shuts down cleanly.

It uses local telemetry only. Export is disabled so you can understand the SDK before adding infrastructure.

## Install

```bash
npm install @varsilias/corelens
```

Install a framework only when you use its adapter:

```bash
npm install express
npm install fastify
npm install hono
npm install @nestjs/common @nestjs/core @nestjs/platform-express reflect-metadata rxjs
```

Express, Fastify, Hono, and Nest runtime packages are optional. Corelens can run without a framework adapter.

## Create a Corelens instance

```ts
import { corelens } from '@varsilias/corelens';

const lens = corelens({
  serviceName: 'checkout-api',
  logs: {
    enabled: true,
    enrichWithTraceContext: true,
  },
  metrics: {
    enabled: true,
  },
  traces: {
    enabled: true,
    samplingRate: 0.25,
  },
  export: {
    enabled: false,
  },
});
```

`serviceName` should be stable. It is how logs, metrics, traces, and exported records identify the service.

## Write a structured log

```ts
lens.logger.info('service started', { port: 3000 });
```

Logs are structured events. Keep values safe to store and search. Do not put secrets, tokens, or full payloads in log context.

## Record a metric

```ts
const requestsTotal = lens.metrics.counter(
  'checkout_requests_total',
  'Total checkout requests',
);

requestsTotal.inc({ route: '/checkout', method: 'POST' });
```

Metrics are stored in a local registry. You can render that registry as Prometheus text or send it through the export pipeline.

## Create a span

```ts
await lens.tracer.withSpan('checkout.create', async (span) => {
  span?.setAttribute('checkout.flow', 'standard');
});
```

Spans explain work that happens inside a request, job, or operation. Use them for meaningful application steps, not every function call.

## Shut down cleanly

```ts
process.on('SIGTERM', async () => {
  await lens.shutdown();
  process.exit(0);
});
```

Shutdown matters when export is enabled. It gives batched logs, metrics, and traces time to flush before the process exits.

## Complete example

```ts
import { corelens } from '@varsilias/corelens';

const lens = corelens({
  serviceName: 'checkout-api',
  logs: {
    enabled: true,
    enrichWithTraceContext: true,
  },
  metrics: {
    enabled: true,
  },
  traces: {
    enabled: true,
    samplingRate: 0.25,
  },
  export: {
    enabled: false,
  },
});

lens.logger.info('service started', { port: 3000 });

const requestsTotal = lens.metrics.counter(
  'checkout_requests_total',
  'Total checkout requests',
);

requestsTotal.inc({ route: '/checkout', method: 'POST' });

await lens.tracer.withSpan('checkout.create', async (span) => {
  span?.setAttribute('checkout.flow', 'standard');
});

process.on('SIGTERM', async () => {
  await lens.shutdown();
  process.exit(0);
});
```
