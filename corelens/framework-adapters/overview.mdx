Framework adapters add HTTP metrics and server spans to supported Node.js frameworks.

Adapters are explicit. Register them where you create the app so the instrumentation boundary is visible.

## Entry points

Express, Fastify, and Hono adapters are exported from `@varsilias/corelens/adapter`.

NestJS support is exported from `@varsilias/corelens/adapter/nest` because it depends on Nest runtime packages such as `@nestjs/common`. Non-Nest services do not need to install Nest dependencies.

## Verification

After registering an adapter:

1. Send a request to a normal route.
2. Check HTTP metrics through your Prometheus endpoint or export destination.
3. Check traces for a server span.
4. Confirm noisy routes such as `/health` and `/metrics` are ignored if configured.

## Common mistakes

- Registering adapters before creating the Corelens instance.
- Enabling HTTP metrics in the adapter but not in `metrics.http`.
- Enabling HTTP tracing in the adapter but not in `traces.http`.
- Importing NestJS APIs from `@varsilias/corelens/adapter` instead of `@varsilias/corelens/adapter/nest`.
- Exposing `/debug/stats` publicly.
