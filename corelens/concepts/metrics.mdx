Metrics show aggregate behavior over time. They are the fastest way to answer questions like “is traffic rising?”, “is latency getting worse?”, and “is this process healthy?”.

Corelens records metrics in a local registry. You can render that registry as Prometheus text or export metrics through the configured export pipeline.

## Counters

Use counters for values that only increase.

```ts
const ordersTotal = lens.metrics.counter(
  'orders_total',
  'Total orders created',
);

ordersTotal.inc({ status: 'created' });
```

## Histograms

Use histograms for values where distribution matters, such as latency or order value.

```ts
const orderValue = lens.metrics.histogram(
  'order_value',
  'Order value in major currency units',
  { buckets: [10, 25, 50, 100, 250, 500] },
);

orderValue.observe(42, { currency: 'USD' });
```

## HTTP metrics

HTTP metrics are recorded through framework adapters. Ignore routes that add noise or create scrape loops.

```ts
metrics: {
  enabled: true,
  http: {
    enabled: true,
    ignoredRoutes: ['/metrics', '/health'],
    buckets: [0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1],
  },
}
```

## Runtime metrics

Runtime metrics help you detect memory growth, event loop pressure, and process-level health.

```ts
metrics: {
  enabled: true,
  runtime: {
    enabled: true,
    intervalMs: 5000,
  },
}
```

Runtime collection uses an interval. Call `lens.shutdown()` during process shutdown so lifecycle cleanup is predictable.

## Prometheus text export

Corelens can render the current metric snapshot as Prometheus text.

```ts
import { PrometheusTextExporter } from '@varsilias/corelens';

const prometheus = new PrometheusTextExporter();

app.get('/metrics', (_req, res) => {
  res.type('text/plain');
  res.send(prometheus.render(lens.metrics.snapshot()));
});
```

For Fastify, Hono, or NestJS, return the rendered text with `content-type: text/plain`.

## Cardinality protection

Use `maxSeriesPerMetric` to protect the process from unbounded metric series.

```ts
metrics: {
  enabled: true,
  maxSeriesPerMetric: 1000,
}
```

Avoid putting user IDs, emails, order IDs, or unbounded strings in metric labels.
