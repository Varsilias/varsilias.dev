Traces show the path of one unit of work. In Corelens, that usually means an HTTP request, a background job, or an outbound call to another service.

Use traces when you need to answer:

- where did this request spend time?
- which operation failed?
- did this service call another service?
- which logs belong to this request?

## Server spans

Framework adapters create server spans for incoming HTTP requests. Enable HTTP tracing in the service configuration, then register the adapter for your framework.

```ts
const lens = corelens({
  serviceName: 'orders-api',
  traces: {
    enabled: true,
    samplingRate: 0.25,
    http: {
      enabled: true,
      ignoredRoutes: ['/health'],
    },
  },
});
```

See [Framework adapters](/corelens/framework-adapters/overview) for Express, Fastify, and Hono setup.

## Manual spans

Use manual spans for important application work inside a request or job.

```ts
await lens.tracer.withSpan('orders.reserveInventory', async (span) => {
  span?.setAttribute('component', 'inventory');
  span?.addEvent('inventory.reserve.started');
});
```

Span names should describe domain operations. Prefer `orders.reserveInventory` over `runFunction`.

## Outbound client spans

`withClientSpan` creates a client span and returns a W3C `traceparent` header. Forward that header to the downstream service so traces can connect across service boundaries.

```ts
const response = await lens.tracer.withClientSpan(
  {
    name: 'GET catalog',
    attributes: {
      'http.method': 'GET',
      'http.url': 'https://catalog.internal/recommendations',
    },
  },
  async ({ traceparent, span }) => {
    const response = await fetch('https://catalog.internal/recommendations', {
      headers: { traceparent },
    });

    span?.setAttribute('http.status_code', response.status);
    return response;
  },
);
```

## Sampling

Sampling controls how many traces Corelens records. `samplingRate` must be between `0` and `1`.

```ts
traces: {
  enabled: true,
  samplingRate: 0.1,
}
```

Use a higher rate while rolling out instrumentation. Lower it for high-traffic production services after you know which spans are useful.

<Warning>
  Sampling reduces trace volume. Metrics should still cover aggregate behavior, because sampled traces do not represent every request.
</Warning>

## Logs and trace correlation

When `logs.enrichWithTraceContext` is enabled, logs written inside an active span can include trace context. That makes it easier to move from a trace to the related logs.

## Common mistakes

- Creating spans for every small function.
- Adding secrets or full payloads as span attributes.
- Forgetting to forward `traceparent` on outbound calls.
- Sampling too aggressively before instrumentation is understood.
