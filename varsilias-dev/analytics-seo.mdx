This page tracks the production setup for search visibility and documentation analytics.

## Current setup

Varsilias.dev includes:

- global SEO metadata in `docs.json`
- page-level `title`, `description`, and selected `keywords`
- `robots.txt`
- `sitemap.xml`
- `llms.txt`
- `humans.txt`
- Google Analytics 4
- Google Search Console verification
- cookie consent gating for analytics telemetry

## Fill in Google Analytics 4

Create a GA4 web stream for `https://www.varsilias.dev`, then replace the placeholder in `docs.json`:

```json
"integrations": {
  "ga4": {
    "measurementId": "G-XXXXXXXXXX"
  }
}
```

Use your real Measurement ID. It starts with `G-`.

## Cookie consent

Analytics is gated behind consent. Mintlify checks this local storage value before enabling telemetry:

```json
"integrations": {
  "cookies": {
    "key": "varsilias_cookie_consent",
    "value": "granted"
  }
}
```

The global `cookie-consent.js` file shows a small banner on first visit. It stores one of these values:

| Value | Meaning |
|-------|---------|
| `granted` | Analytics may run |
| `denied` | Analytics stays disabled |

If someone changes their preference, the page reloads so Mintlify can re-check the consent value before loading telemetry.

## Fill in Google Search Console

Add `https://www.varsilias.dev` as a URL-prefix property in Google Search Console. Choose the HTML tag verification method and copy only the `content` value.

Replace this placeholder in `docs.json`:

```json
"seo": {
  "metatags": {
    "google-site-verification": "GOOGLE_SEARCH_CONSOLE_VERIFICATION_TOKEN"
  }
}
```

Do not paste the full `<meta>` tag. Paste only the token.

## Submit the sitemap

After deployment, open Google Search Console and submit:

```text
https://www.varsilias.dev/sitemap.xml
```

Mintlify can generate a sitemap automatically, but this project also keeps a root-level `sitemap.xml` so the current public URL set is explicit.

## Maintenance checklist

- Add new public pages to `docs.json`.
- Add new public pages to `sitemap.xml`.
- Keep page descriptions specific.
- Keep `llms.txt` updated for major documentation entry points.
- Re-submit the sitemap after major namespace changes.
- Keep consent copy accurate if analytics tools change.
