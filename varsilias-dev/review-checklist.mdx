Use this checklist before merging documentation changes.

## Accuracy

- The page describes current or clearly intended behavior.
- Provisional APIs are marked as provisional.
- Commands and examples are runnable or clearly scoped.
- Links point to existing pages.

## Structure

- The page starts with context before steps.
- Sections are short enough to scan.
- Navigation cards appear at the end of overview or quickstart pages.
- Related content links to the right namespace.

## Style

- Headings use sentence case.
- Sentences stay concise.
- Code blocks have language identifiers.
- UI labels are bolded when referenced.
- File paths and commands use inline code.
