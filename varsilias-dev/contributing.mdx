Contributions should improve how developers understand, install, operate, or extend a tool.

## Good first contributions

- Fix unclear setup instructions.
- Add a missing prerequisite.
- Improve a troubleshooting symptom.
- Add a tested example.
- Clarify a tradeoff in architecture docs.

## Before you edit

1. Choose one tool namespace.
2. Read the overview and nearby pages.
3. Check whether the change belongs in a product page or the Varsilias.dev contribution docs.
4. Keep the pull request focused.

## Pull request checklist

- The page has frontmatter with `title` and `description`.
- Internal links use root-relative paths.
- Code blocks include a language.
- New pages are registered in `docs.json`.
- The change avoids marketing language.
